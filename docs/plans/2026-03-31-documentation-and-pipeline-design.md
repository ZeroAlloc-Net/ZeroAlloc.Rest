# ZeroAlloc.Rest — Documentation & Pipeline Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Bring ZeroAlloc.Rest to full ZeroAlloc ecosystem parity — comprehensive documentation, CI/CD pipelines, versioning, and NuGet publishing infrastructure matching ZeroAlloc.Mediator conventions exactly.

**Architecture:** Full Mediator-parity approach. GitVersion (ContinuousDeployment) for pre-release alpha builds on every main push; release-please for stable versioning. Docusaurus-compatible `docs/` folder (pulled as submodule by `ZeroAlloc-Net/.website`). All NuGet metadata centralised in `Directory.Build.props`.

**Tech Stack:** GitHub Actions, GitVersion 6.x (dotnet tool), release-please v4, Conventional Commits, Docusaurus frontmatter (for website integration), shields.io badges, NuGet.

## Conventions (matching ZeroAlloc.Mediator exactly)

- **Domain:** `rest.zeroalloc.net` (PackageProjectUrl)
- **NuGet packages:** ZeroAlloc.Rest, ZeroAlloc.Rest.Attributes, ZeroAlloc.Rest.Generator, ZeroAlloc.Rest.SystemTextJson, ZeroAlloc.Rest.Tools
- **Version:** injected at build time via `-p:Version=`; no `<Version>` in any `.csproj`
- **Badges:** NuGet (ZeroAlloc.Rest) + CI badge + MIT badge — in that order
- **Analyzers in Directory.Build.props:** Meziantou, Roslynator, ErrorProne.NET.CoreAnalyzers, ErrorProne.NET.Structs, NetFabric.Hyperlinq.Analyzer, ZeroAlloc.Analyzers — all `PrivateAssets=all`
- **Docs frontmatter:** `id`, `title`, `slug`, `description`, `sidebar_position` on every doc page
- **CHANGELOG:** auto-generated by release-please; never manually edited
- **Website trigger:** `WEBSITE_DISPATCH_TOKEN` secret; sends `submodule-update` event to `ZeroAlloc-Net/.website`
- **NuGet secret:** `NUGET_API_KEY` secret in GitHub Actions
